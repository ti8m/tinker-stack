= Prototyp
:experimental:

== Zweck

Der Starter-Prototyp ist absichtlich minimal. Er dient als:

* Einstiegspunkt für den ersten validierbaren User Flow
* Ort für schnelle UX-Iterationen
* Schale, in die eigene API- und Mock-Logik eingesetzt wird

== Technischer Aufbau

* React Router SPA mit TypeScript
* Tailwind CSS
* Lokales MSW-Bootstrapping
* Starter-Landing-Page statt Demo-Workflow

== Startpunkt

Der Prototyp enthält standardmässig:

* eine Root-Layout-Datei
* eine Landing-Route
* MSW Pass-Through Handler

Von dort aus kann die gewünschte Feature-Struktur aufgebaut werden.

== Optionale Beispiele

Wenn ein Beispiel generiert wurde, enthält dieses einen vollständigen Referenz-Flow in seinem
eigenen Verzeichnis unter `examples/<name>/`.

Diese Beispiele gehören nicht zur Build-Pipeline des Root-Projekts und können unabhängig gestartet
oder gelöscht werden.
