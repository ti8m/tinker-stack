== Voraussetzungen

Für die Entwicklung werden folgende Tools benötigt:

* Node.js >= 22
* npm >= 10
* Git

== Repository klonen

[source,bash]
----
git clone TODO
cd planning-stack-template
----

== Installation

Das Projekt verwendet npm Workspaces. Alle Abhängigkeiten können mit einem Befehl installiert werden:

[source,bash]
----
npm install
----

== Entwicklungsserver starten

Der Entwicklungsserver kann direkt im Root-Verzeichnis gestartet werden:

[source,bash]
----
npm run dev
----

Dies startet den Prototypen und die Watcher für die abhängigen Pakete.

Die Anwendung ist dann unter http://localhost:5173 erreichbar.

== Mock-Daten

Der Starter liefert absichtlich leere Mock-Daten. Sobald eigene Generatoren vorhanden sind, kann das
JSON wie gewohnt gebaut werden:

[source,bash]
----
npm run build:data
----

== Optionale Beispiele

Falls das Projekt mit einem Beispiel generiert wurde, liegt dieses unter `examples/<name>/`.

Die Beispiele sind unabhängig vom Hauptprojekt:

* Sie sind nicht Teil der Root-Workspaces
* Sie werden nicht automatisch von Turborepo gebaut
* Sie müssen im jeweiligen Beispielordner separat installiert werden

== Projektstruktur

[source]
----
.
├── api/             # Domain Types & Enums
├── config/
│   ├── eslint/      # ESLint Konfigurationen
│   ├── typescript/  # Typescript Konfigurationen
├── docs/            # Dokumentation
├── frontend/        # Platzhalter für das produktive Frontend
├── mocks/           # Starter für generierte Mock-Daten
├── msw/             # Starter für MSW-Handler
├── prototype/       # Klickbarer UX-Prototyp
└── ui/              # Design-System Komponenten
----

== Typische Entwicklungsabläufe

=== Erste Produktstrecke im Prototyp entwickeln

1. Feature-Branch erstellen
2. Route in `prototype/app/routes.ts` ergänzen
3. UI im Prototyp aufbauen
4. Falls nötig Mock-Daten und MSW-Handler ergänzen
5. Validieren und iterieren

=== Optionales Beispiel untersuchen

1. In den entsprechenden `examples/<name>/` Ordner wechseln
2. Dort `npm install` ausführen
3. Beispiel lokal starten oder bauen
4. Gewünschte Muster gezielt in das Hauptprojekt übernehmen
