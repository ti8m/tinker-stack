= Getting Started
:experimental:
:icons: font

== Prerequisites

The following tools are required for development:

* Node.js >= 22
* npm >= 10
* Git

== Clone the repository

[source,bash]
----
git clone TODO
cd planning-stack-template
----

== Installation

The project uses npm Workspaces. All dependencies can be installed with a single command:

[source,bash]
----
npm install
----

== Set up the development environment

=== Generate mock data

Before the first start, build the mock data:

[source,bash]
----
npm run build:data
----

=== Start the development server

The development server can be started from the root directory:

[source,bash]
----
npm run dev
----

This starts:

* The prototype development server
* A watcher for mock data
* A watcher for UI components

The application is available at http://localhost:5173.

== Optional examples

If the project was generated with an example, it is located under `examples/<name>/`.

Examples are independent from the main project:

* They are not part of the root workspaces
* They are not automatically built by Turborepo
* They must be installed separately inside their own folder

== Project structure

[source]
----
.
├── api/             # Domain types & enums
├── config/
│   ├── eslint/      # ESLint configurations
│   ├── typescript/  # TypeScript configurations
├── docs/            # Documentation
├── frontend/        # Placeholder for the production frontend
├── mocks/           # Starter for generated mock data
├── msw/             # Starter for MSW handlers
├── prototype/       # Clickable UX prototype
└── ui/              # Design system components
----

== Typical development workflows

=== Develop a new feature in the prototype

1. Create a feature branch
2. Add a route in `prototype/app/routes.ts`
3. Build the UI using shadcn/ui components
4. Extend mock data in `mocks/` if needed
5. Test and iterate
6. Open a pull request

=== Explore an optional example

1. Navigate into the `examples/<name>/` folder
2. Run `npm install` there
3. Start or build the example locally
4. Copy the patterns you need into the main project

== Troubleshooting

[qanda]
TypeScript errors after `npm install`::
  Run `npm run build` to compile all packages first.

Mock data is not updated::
  Delete the `.cache` folder and re-run `npm run build:data`.
