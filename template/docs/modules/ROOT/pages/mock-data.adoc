= Testdaten
:experimental:

== Überblick

Das `@repo/mocks` Package ist im Starter absichtlich neutral gehalten.

Die Default-Implementierung erzeugt leere Datensätze, damit `npm run build:data` sofort
funktioniert, auch wenn noch keine Domänenmodelle definiert wurden.

== Starter-Verhalten

* Presets für `small`, `medium` und `full` sind vorhanden
* Der Generator liefert standardmässig ein leeres JSON-Objekt
* Die Struktur kann später durch projektspezifische Generatoren ersetzt werden

== Nächste Schritte

1. Eigene Domänentypen in `@repo/api` definieren
2. Presets in `mocks/src/generators/config.ts` anpassen
3. Datenlogik in `mocks/src/generators/dataset.ts` implementieren
4. JSON mit `npm run build:data` erzeugen

== Beispiele

Wenn ein optionales Beispiel mitgeneriert wurde, zeigt dessen `mocks/` Package eine vollständige
Umsetzung in einem unabhängigen Kontext.
