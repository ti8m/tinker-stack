= Monorepo

Siehe auch README.md.

Wir verwenden Turborepo mit kompilierten internen Paketen.

Alle Pakete sind ESM-Module und verwenden den `@repo` Scope.

== Apps und Pakete

- `@repo/frontend`: Platzhalter für das produktive Frontend
- `@repo/prototype`: minimaler Prototyp für frühe UX-Validierung
- `@repo/api`: Domain-Typen und Enums
- `@repo/docs`: Antora-Dokumentation
- `@repo/mocks`: Starter für generierte Mock-Daten
- `@repo/msw`: Starter für MSW-Handler
- `@repo/ui`: gemeinsame UI-Komponenten
- `@repo/eslint-config`: `eslint`-Konfigurationen
- `@repo/typescript-config`: gemeinsame `tsconfig.json`

== Build

Die Basis-Workspaces werden über Turborepo gebaut:

```
npm run build
```

Weitere hilfreiche Kommandos:

```
npm run build:data
npm run build:msw
npm run build:prototype
```

== Entwicklung

Zum Starten der Basis-Entwicklung:

```
npm run dev
```

== Optionale Beispiele

Beispiele werden unter `examples/<name>/` erzeugt und absichtlich nicht in die Root-Workspaces
aufgenommen.

Dadurch:

* ignoriert Turborepo sie standardmässig
* können sie separat installiert werden
* können sie jederzeit gelöscht werden
