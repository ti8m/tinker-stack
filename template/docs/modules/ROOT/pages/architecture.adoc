== Überblick

Das PLANNING STACK TEMPLATE Frontend ist als Monorepo aufgebaut. Der Fokus liegt auf:

* schneller Iteration
* klar getrennten Paketen
* Typsicherheit durch TypeScript
* optionalen, unabhängigen Referenzbeispielen

== Architekturprinzipien

=== Starter zuerst

Das Basisprojekt soll sofort installierbar und buildbar sein, ohne eine vorgefertigte Demo zu
erzwingen.

=== Prototypen-getriebene Entwicklung

Der empfohlene Ablauf bleibt:

1. Feature zuerst im Prototyp modellieren
2. Feedback einholen
3. Validierte Muster in die produktive Anwendung übernehmen

=== Unabhängige Beispiele

Optionale Beispiele liegen ausserhalb der Root-Workspaces in `examples/<name>/`.

Das bedeutet:

* keine automatische Teilnahme an `npm run build`
* keine Kopplung an die Root-Turbo-Pipeline
* sichere Löschbarkeit ohne Folgeschäden im Hauptprojekt

=== Mocking

[plantuml,msw,svg]
----
@startuml
actor Browser
participant "Prototype" as App
participant "MSW" as MSW

Browser -> App: Request
App -> MSW: Optional API Call
MSW --> App: Mocked or passthrough response
App --> Browser: Render
@enduml
----

Die Basis liefert nur eine starter-sichere MSW-Schicht. Projektspezifische Handler und Datensätze
werden später ergänzt oder aus einem optionalen Beispiel übernommen.
