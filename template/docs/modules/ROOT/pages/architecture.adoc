== Overview

The PLANNING STACK TEMPLATE frontend is structured as a monorepo. The focus is on:

* fast iteration
* clearly separated packages
* type safety through TypeScript
* optional, independent reference examples

== Architecture principles

=== Starter first

The base project should be immediately installable and buildable without forcing a pre-built demo.

=== Prototype-driven development

The recommended workflow remains:

1. Model the feature in the prototype first
2. Gather feedback
3. Carry validated patterns into the production application

=== Independent examples

Optional examples live outside the root workspaces under `examples/<name>/`.

This means:

* no automatic participation in `npm run build`
* no coupling to the root Turbo pipeline
* safe to delete without side effects in the main project

=== Mocking

[plantuml,msw,svg]
----
@startuml
actor Browser
participant "Prototype" as App
participant "MSW" as MSW

Browser -> App: Request
App -> MSW: Optional API Call
MSW --> App: Mocked or passthrough response
App --> Browser: Render
@enduml
----

The base provides only a starter-safe MSW layer. Project-specific handlers and datasets are added later or copied from an optional example.
